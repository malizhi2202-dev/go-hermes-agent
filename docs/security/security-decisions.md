# Go 迁移安全决策

## 已去掉的高风险能力

### 1. 任意命令执行

Python 版本的 `terminal` 工具具备很强能力，但也具备明显风险：

- 命令注入
- 文件破坏
- 横向移动
- 凭据泄漏

Go 版本当前不实现这条能力链。

### 2. 任意代码执行

Python 版本的 `execute_code` 可运行脚本。Go 版本当前不保留。

原因：

- 恶意代码注入面过大
- 沙箱、资源限制、网络隔离、审计未完成前不应迁移

### 3. 未受限动态插件 / 工具注入

Python 版存在 MCP、plugin、skill script 等动态扩展面。Go 版本暂不直接接入。

## 已新增的安全机制

### 1. 本地账号登录

- bcrypt 密码哈希
- 最小密码长度
- 登录失败次数限制
- 登录窗口限流

### 2. JWT 会话鉴权

- 服务端签发令牌
- 带过期时间
- Bearer Token 校验

### 3. 受控配置

- 显式配置文件加载
- API key 来自环境变量
- 不把明文密钥写入数据库

### 4. SQLite 落盘控制

- 用户、会话、审计事件统一进入受控数据库
- 数据目录与服务目录分离

## 保留但需后续强化的点

- JWT 目前采用本地 secret 文件，后续可接 KMS / Vault
- 登录失败限制目前基于数据库计数，后续可叠加 IP 维度
- LLM provider 当前是 OpenAI 兼容接口，后续可补多 provider 抽象

## 返库人工审核项

以下 Python 特性若必须在 Go 里恢复，必须先人工审核：

1. 终端执行
2. 浏览器自动化
3. 子代理委派
4. 动态技能脚本
5. 插件热加载
